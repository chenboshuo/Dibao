# Dibao 许可证 FAQ

Dibao / 邸报 使用 Business Source License 1.1（BUSL-1.1），并通过 Additional Use Grant 实现 source-available / fair-code / delayed open source 模式。每个公开发布版本在自己的 Change Date 后自动变更为 Apache License 2.0。

## 邸报现在是什么许可证？

当前版本使用 Business Source License 1.1，SPDX identifier 为 `BUSL-1.1`。Change License 是 Apache License 2.0，SPDX identifier 为 `Apache-2.0`。

## 它还是 fair-code 吗？

是。邸报仍然是 fair-code / source-available / 可自托管项目，只是从自定义许可证改为用 BUSL-1.1 实现这种模式。

## 它是开源吗？

当前版本在 Change Date 前不是 OSI 意义上的开源。到 Change Date 后，该具体版本会自动按 Apache-2.0 开源。

## 我可以自己部署吗？

可以。个人、家庭、研究、评估、学习和非商业用途可以免费自托管和生产使用。

## 公司内部可以用吗？

可以。公司、机构、学校、非营利组织或其他组织可以在自身内部业务中免费使用、修改、自托管和生产使用。

## 我可以修改吗？

可以。你可以为了被 BUSL-1.1 和 Additional Use Grant 允许的用途修改 Dibao。

## 我可以 fork 吗？

可以。你可以 fork、学习、研究、修改和非商业分发，但必须保留 Dibao 来源说明、版权声明和许可证声明。

## 我可以给客户收费部署吗？

可以，但客户获得的必须是部署在客户自有环境、客户自有账号或客户控制基础设施中的 Dibao 实例。你不能把 Dibao 做成自己的托管平台、白标产品、转售产品或竞争性 SaaS，也不能暗示自己提供官方商业版本。

## 我可以提供收费 SaaS 吗？

不可以，除非获得 Licensor 的单独商业授权。收费托管版、SaaS、Managed Service、Cloud Service 或类似服务都需要商业授权。商业授权联系：https://dibao.app。

## 我可以做白标版吗？

不可以，除非获得 Licensor 的单独商业授权。商业授权联系：https://dibao.app。

## 我可以移除 Dibao 名称和版权声明吗？

不可以。未经授权，不得移除或隐藏 Dibao 名称、版权声明、许可证声明或必要来源说明。

## 4 年后会发生什么？

该具体发布版本会自动转为 Apache-2.0。这个转换不需要额外操作；到 Change Date 后自动生效。

## 新版本也会自动开源吗？

会，但每个版本分别计算自己的 Change Date。新版本不会因为旧版本到期而提前转为 Apache-2.0。

## main 分支怎么算？

`main` 是开发版，可能包含 TODO 日期或待发布参数。具体 Release Date、Change Date 和法律快照以对应 release tag 中冻结的 `LICENSE.md` 为准。

## Release Date 是什么？

Release Date 是某个具体版本第一次公开发布的日期。它不是项目级固定日期，而是每个版本分别确定的许可证参数。`main` 分支可以保留 `TODO`，正式发布 `vX.Y.Z` 时再在该 tag 的 `LICENSE.md` 中冻结 Release Date，并据此计算 Change Date。

## 官方以后能不能做商业托管或商业版？

可以。项目维护者保留提供官方商业实例、商业托管、商业授权、闭源增值服务和企业版功能的权利。

## 外部贡献会不会被用于商业版？

会。提交 PR 即表示你同意 [CONTRIBUTING.md](../CONTRIBUTING.md) 中的贡献授权条款。你仍保留自己贡献的版权，同时授予项目维护者永久、全球、非排他、免费、免版税、不可撤销的使用、复制、修改、分发、再许可和重新许可权利。
