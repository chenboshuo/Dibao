# 邸报 Dibao 文章行为动作交互微规格 v0

## 文档目的

本文定义邸报 MVP 中用户如何对文章执行行为动作，以及这些动作如何成为后续个性化推荐的信号。

本文用于指导前端实现，不替代：

- [API Contract](./api-contract.md)：文章 action 的接口和状态字段。
- [低保真页面线框](./wireframes.md)：文章列表和阅读页布局骨架。
- [视觉与交互设计规范 v0](./visual-interaction-spec-v0.md)：全局视觉、反馈、可访问性和状态规范。
- [Profile Algorithm v0](./profile-algorithm-v0.md)：行为权重、画像更新资格和推荐排序规则。

## 范围

覆盖动作：

- 列表滚过未点进：`impression`
- 打开文章：`open`
- 收藏 / 取消收藏：`favorite`
- 稍后读 / 移出稍后读：`read_later`
- 隐藏：`hide`
- 不感兴趣：`not_interested`
- 阅读进度：`read_progress`

不进入 MVP：

- 多用户、团队、共享、评论、社交动作。
- 标签系统、文件夹式文章整理、复杂归档系统。
- 批量动作。
- 手动“标记已读 / 标记未读”按钮。`mark_read` / `mark_unread` 只作为 API 兼容项保留，不是当前 Web 的主交互。
- Undo 恢复机制。
- 隐藏文章管理页或不感兴趣管理页。
- 复杂推荐设置页。

## 基本原则

- 阅读优先：动作入口要清楚，但不能压过标题、来源、摘要和正文。
- 显式优先：用户明确点击的动作比自动推断更可信。
- 反馈克制：成功反馈主要通过按钮状态和列表变化表达，不用大 toast 堆叠。
- 失败可恢复：乐观更新失败时必须回滚到服务端前状态。
- 推荐可调教：收藏、稍后读、隐藏、不感兴趣都应成为推荐系统可解释、可调的输入。
- 弱信号保守：打开和阅读进度是隐式信号，不应被解释成强烈偏好或强烈厌恶。

## 数据与 API 对齐

所有文章动作统一调用：

```text
POST /api/articles/:id/actions
```

请求形态：

```json
{
  "type": "favorite",
  "value": true,
  "metadata": {}
}
```

响应中的 `state` 是前端更新文章状态的权威结果：

```ts
state: {
  read: boolean;
  favorited: boolean;
  readLater: boolean;
  hidden: boolean;
  notInterested: boolean;
  readingProgress: number;
  interactionStatus: "unseen" | "seen" | "saved" | "ignored" | "opened" | "reading" | "read";
  openedAt: number | null;
  ignoredAt: number | null;
}
```

前端可以做乐观更新，但最终必须以响应 state 对齐。

## 动作总览

| 动作 | API type | value | 信号类型 | 推荐意义 | 当前列表影响 |
| --- | --- | --- | --- | --- | --- |
| 列表滚过未点进 | `impression` | 可省略或 `true` | 隐式行为 | 仅对完全未触达文章是轻度负信号 | 不移除，视觉弱化 |
| 打开文章 | `open` | 可省略或 `true` | 隐式行为 | 轻度正信号，只记录点进 | 不移除 |
| 收藏 | `favorite` | `true` | 显式意图 | 强正反馈，增强正向兴趣 | 收藏列表中出现 |
| 取消收藏 | `favorite` | `false` | 显式意图 | stats only，弱负纠正 | 收藏列表中移除 |
| 稍后读 | `read_later` | `true` | 显式意图 | 中度正反馈 | 稍后读列表中出现 |
| 移出稍后读 | `read_later` | `false` | 显式意图 | stats only，弱负纠正 | 稍后读列表中移除 |
| 隐藏 | `hide` | `true` | 显式意图 | 负反馈，弱于不感兴趣 | 普通列表中移除 |
| 不感兴趣 | `not_interested` | `true` | 显式意图 | 强负反馈，增强负向兴趣 | 普通列表中移除 |
| 阅读进度 | `read_progress` | `0.0-1.0` | 隐式行为 | 进度达阈值后成为正向阅读信号 | 不移除 |

说明：

- “普通列表”指推荐、最新、搜索结果等以文章发现和阅读为主的列表。
- 收藏列表和稍后读列表中的移除动作只影响当前对应列表，不代表删除文章。
- `hidden` 与 `notInterested` 默认不在普通列表显示。
- `unseen` 是新文章默认状态，表示尚未发生任何行为，也是未读筛选和未读计数的唯一来源。
- `seen` 表示发生过中性行为但没有保存、打开、阅读或忽略；`saved` 表示收藏、点赞或稍后读至少一个为真。`seen/saved` 都会移出未读，但不等于已读。
- `ignored` 只表示完全未触达文章被列表划过：没有打开、收藏、点赞、稍后读、阅读进度或其他历史行为。收藏、稍后读、点赞、打开、阅读后的文章即使再次被划过，也不得变成 `ignored`，不得产生划过负分。
- `opened` 表示已点进但尚未形成足够阅读深度；`reading/read` 由 `read_progress` 派生。被忽略文章再次点进后状态回到 `opened`，后续通过 `read_progress` 判断真实阅读深度。
- 色条语义以状态派生为准：`unseen` 显示未读深松色条；`saved/opened/reading/read` 不显示未读色条，但可以叠加收藏黄条、稍后读绿条；`ignored` 是极淡弱化条，且与收藏/稍后读条互斥。

## 触发位置

### 文章列表 ArticleItem

列表项保留高密度结构：

```text
Meta Row: 来源 · 时间 · 状态
Title Row: 标题
Summary Row: 摘要
Action Row: 收藏 / 稍后读 / 不感兴趣 / 为什么 / 更多
```

列表中直接暴露：

- 收藏 / 取消收藏。
- 稍后读 / 移出稍后读。
- 不感兴趣。
- 推荐解释入口。

列表中可放入更多菜单：

- 标记已读。
- 标记未读。
- 隐藏。
- 打开原文。

交互规则：

- 点击标题或列表主体触发打开文章。
- 点击动作按钮不得触发打开文章。
- 移动端动作按钮常显。
- 桌面端动作按钮可以常显或在 hover / focus-within 时增强显示，但键盘和触摸必须可达。

### 文章详情 Reader

阅读页顶部工具条：

- 返回。
- 原文。
- 阅读设置。

阅读页底部或正文后动作区：

- 收藏 / 取消收藏。
- 稍后读 / 移出稍后读。
- 不感兴趣。
- 标记已读 / 标记未读。

更多菜单：

- 隐藏。
- 打开原文。

自动触发：

- 进入阅读页发送 `open`。
- 滚动或停留过程中节流发送 `read_progress`。

### 快捷按钮

快捷按钮只覆盖高频动作：

- 收藏。
- 稍后读。
- 不感兴趣。
- 标记已读 / 未读。

快捷按钮要求：

- 图标按钮必须有 `aria-label`。
- 不熟悉或负向动作使用文字或图标 + 文本。
- 移动端触摸目标不小于 44px。
- 按钮状态变化不能造成列表项高度跳动。

## 单个动作规格

### 打开文章

触发位置：

- 点击文章列表主体或标题。
- 从搜索、收藏、稍后读、推荐解释关联入口进入文章详情。

API：

```json
{ "type": "open", "value": true, "metadata": {} }
```

信号分类：

- 隐式行为信号。
- 轻度正信号。
- 不直接创建兴趣簇。

视觉反馈：

- 不显示成功提示。
- 进入阅读页后文章标题和正文正常加载。
- 若详情加载失败，显示阅读页错误状态并提供返回列表。

当前列表影响：

- 不立即从列表移除。
- 不自动标记已读。
- 可在本地记录“已打开”视觉弱化，但必须与 `read` 状态区分。

推荐影响：

- 进入行为统计。
- 可影响来源统计和短期排序。
- 不能被解释成强兴趣。

### 标记已读

> Legacy / API compatibility：当前 Web 不提供“标记已读”按钮。真实阅读完成由 `read_progress` 和阅读时长派生；本节仅保留给旧客户端或未来批量管理能力参考。

触发位置：

- 文章列表更多菜单。
- 阅读页动作区。
- 后续批量动作不进入 MVP。

API：

```json
{ "type": "mark_read", "value": true, "metadata": {} }
```

信号分类：

- 显式用户意图。
- stats only。

视觉反馈：

- 乐观更新 `state.read = true`。
- 标题从未读样式变为已读样式。
- 按钮文案切换为“标记未读”。
- 成功不需要 toast。
- 失败回滚，并显示 `标记已读失败`。

当前列表影响：

- 在 `status=unread` 的列表中成功后立即移除。
- 在 `status=all` 中保留，但视觉弱化。
- 在收藏和稍后读列表中保留，除非当前筛选限定未读。

推荐影响：

- 表示用户处理了文章，但不是强兴趣。
- 不创建正向或负向兴趣簇。

### 标记未读

> Legacy / API compatibility：当前 Web 不提供“标记未读”按钮。文章交互状态主要由 `impression`、`open` 和 `read_progress` 派生。

触发位置：

- 文章列表更多菜单。
- 阅读页动作区。

API：

```json
{ "type": "mark_unread", "value": true, "metadata": {} }
```

信号分类：

- 显式用户意图。
- stats only。

视觉反馈：

- 乐观更新 `state.read = false`。
- 标题恢复未读样式。
- 按钮文案切换为“标记已读”。
- 成功不需要 toast。
- 失败回滚，并显示 `标记未读失败`。

当前列表影响：

- 在未读列表中出现或保留。
- 在已读筛选中成功后移除。
- 在推荐列表中不强制上浮，由下一次排序决定。

推荐影响：

- 视为状态纠正，不作为强偏好。
- 不创建兴趣簇。

### 收藏 / 取消收藏

触发位置：

- 文章列表 Action Row。
- 阅读页动作区。
- 可在推荐解释 Popover / Bottom Sheet footer 中作为可选快捷动作。

API：

```json
{ "type": "favorite", "value": true, "metadata": {} }
```

取消收藏：

```json
{ "type": "favorite", "value": false, "metadata": {} }
```

信号分类：

- 显式用户意图。
- 收藏是强正反馈。
- 取消收藏是弱负纠正，stats only。

视觉反馈：

- 乐观更新收藏按钮 selected 状态。
- 选中态使用克制强调色，不整行变色。
- 按钮 aria-label 在“收藏”和“取消收藏”之间切换。
- 成功不需要 toast。
- 失败回滚，并显示 `收藏失败` 或 `取消收藏失败`。

当前列表影响：

- 收藏成功后文章可出现在收藏列表。
- 在推荐、最新、搜索中不移除。
- 在收藏列表中取消收藏后立即移除当前条目。
- 若收藏列表因此为空，显示收藏空状态。

推荐影响：

- 收藏可创建或增强正向兴趣簇。
- 收藏是最强正向显式反馈之一。
- 取消收藏不应直接创建负向兴趣簇。

### 稍后读 / 移出稍后读

触发位置：

- 文章列表 Action Row。
- 阅读页动作区。

API：

```json
{ "type": "read_later", "value": true, "metadata": {} }
```

移出稍后读：

```json
{ "type": "read_later", "value": false, "metadata": {} }
```

信号分类：

- 显式用户意图。
- 稍后读是中度正反馈。
- 移出稍后读是弱负纠正，stats only。

视觉反馈：

- 乐观更新稍后读按钮 selected 状态。
- 选中态可使用状态标记或轻量图标变化。
- 成功不需要 toast。
- 失败回滚，并显示 `稍后读更新失败`。

当前列表影响：

- 加入稍后读后文章可出现在稍后读列表。
- 在推荐、最新、搜索中不移除。
- 在稍后读列表中移出后立即移除当前条目。
- 若稍后读列表因此为空，显示稍后读空状态。

推荐影响：

- 稍后读增强正向兴趣，但弱于收藏。
- 移出稍后读不创建负向兴趣簇。

### 隐藏

触发位置：

- 文章列表更多菜单。
- 阅读页更多菜单。

API：

```json
{ "type": "hide", "value": true, "metadata": {} }
```

信号分类：

- 显式用户意图。
- 负反馈，弱于不感兴趣。

视觉反馈：

- 触发后按钮进入 busy 状态。
- 成功后从当前普通列表移除。
- 成功不需要 toast；列表变化就是反馈。
- 失败时保留文章并显示 `隐藏失败`。

当前列表影响：

- 在推荐、最新、搜索结果中成功后立即移除。
- 如果在阅读页隐藏，返回上一列表时该文章不再出现。
- 若当前列表因此为空，显示当前视图空状态。
- MVP 不提供隐藏文章管理页。

推荐影响：

- 可增强负向兴趣簇。
- 也可影响来源统计。
- 语义是“不要再给我看这篇”，不是“这个主题我不喜欢”。

### 不感兴趣

触发位置：

- 文章列表 Action Row。
- 阅读页动作区。
- 推荐解释 Popover / Bottom Sheet footer 可提供同一动作。

API：

```json
{ "type": "not_interested", "value": true, "metadata": {} }
```

信号分类：

- 显式用户意图。
- 强负反馈。

视觉反馈：

- 触发后按钮进入 busy 状态。
- 成功后从当前普通列表移除。
- 成功不需要 toast；列表变化就是反馈。
- 失败时保留文章并显示 `不感兴趣操作失败`。
- 负向动作按钮不要使用高饱和红色常显；只在 hover、focus、确认态或错误态使用 danger 色。

当前列表影响：

- 在推荐、最新、搜索结果中成功后立即移除。
- 如果在阅读页触发，可返回上一列表，且该文章不再出现。
- 若当前列表因此为空，显示当前视图空状态。
- MVP 不提供恢复入口。

推荐影响：

- 创建或增强负向兴趣簇。
- 是最强负向显式反馈。
- 后续推荐解释可出现类似 `与不感兴趣主题接近` 的负向原因。

### 阅读进度

触发位置：

- 阅读页正文滚动。
- 页面可见性变化。
- 离开阅读页前 flush 最新进度。

API：

```json
{
  "type": "read_progress",
  "value": 0.72,
  "metadata": {
    "durationMs": 42000
  }
}
```

信号分类：

- 隐式行为信号。
- 达到阈值后可成为正向阅读信号。

记录规则：

- 只记录最高阅读进度。
- 低于已记录最高值时不发送。
- 建议跨过 25%、50%、75% 阈值时发送。
- 未跨阈值时按不小于 5 秒节流发送。
- 页面隐藏、返回、关闭前尽量发送最后一次最高进度。
- 达到 90% 或后端定义的完成条件时，可发送 `read_complete`，该事件不需要单独 UI 入口。

视觉反馈：

- 不显示成功提示。
- 可在文章状态中保留阅读进度，但 MVP 不要求显示进度条。
- 若发送失败，不打断阅读；可静默重试一次或等待下一次进度上报。

当前列表影响：

- 不立即移除文章。
- 后端若根据完成阅读更新 `read`，前端以响应 state 为准。

推荐影响：

- 25% 是弱阅读信号，主要进入统计。
- 50%、75%、读完可增强正向兴趣。
- 短时间低进度离开可由后端派生 quick bounce，但 MVP 前端不直接发送 quick bounce。

## 视觉反馈规则

### 按钮状态

按钮至少支持：

- default：可操作。
- selected：收藏、稍后读等布尔状态已开启。
- busy：请求中，禁止重复触发。
- disabled：当前状态不可操作或数据未加载。
- error：短暂错误状态，随后恢复 default 或 selected。

状态文案：

- 收藏：`收藏` / `取消收藏`
- 稍后读：`稍后读` / `移出稍后读`
- 已读：`标记已读` / `标记未读`
- 隐藏：`隐藏这篇`
- 不感兴趣：`不感兴趣`

### 成功提示

默认不使用成功 toast。

成功反馈优先级：

1. 按钮 selected 状态变化。
2. 列表项视觉变化。
3. 当前列表移除。
4. 空状态更新。

只有后台耗时动作或用户不易察觉的动作才考虑 notice；本文覆盖的 MVP 动作不需要成功 toast。

### 错误提示

错误提示保持短句：

- `操作失败，请重试。`
- `收藏失败。`
- `稍后读更新失败。`
- `标记已读失败。`
- `隐藏失败。`
- `不感兴趣操作失败。`

错误显示位置：

- 列表项内优先显示 inline notice。
- 阅读页动作区显示 inline notice。
- 不使用全屏错误覆盖已有文章内容。

错误恢复：

- 乐观更新必须回滚。
- busy 状态结束。
- focus 保留在触发按钮或回到合理位置。

## 当前列表与空状态

### 移除规则

成功后立即从当前列表移除：

- `hide`。
- `not_interested`。
- 收藏列表中的取消收藏。
- 稍后读列表中的移出稍后读。
- 未读筛选中的标记已读。
- 已读筛选中的标记未读。

成功后保留在当前列表：

- `open`。
- `read_progress`。
- 推荐、最新、搜索中的收藏。
- 推荐、最新、搜索中的稍后读。
- `status=all` 中的标记已读 / 未读。

### 空状态

列表因动作变空时，使用当前视图对应空状态：

- 推荐：`暂时没有推荐文章`。
- 最新：`暂时没有文章`。
- 收藏：`还没有收藏文章`。
- 稍后读：`还没有稍后读文章`。
- 搜索：`没有匹配结果`。

空状态不解释算法细节，不加入插画，不新增复杂入口。

## 推荐信号分类

### 显式用户意图

这些动作来自用户明确点击：

- `mark_read`
- `mark_unread`
- `favorite`
- `read_later`
- `hide`
- `not_interested`

推荐系统处理原则：

- 显式正反馈优先增强正向兴趣。
- 显式负反馈优先增强负向兴趣。
- 状态纠正动作进入统计，不直接创建相反兴趣。

### 隐式行为信号

这些动作由阅读流程自动产生：

- `open`
- `read_progress`
- `read_complete`

推荐系统处理原则：

- 隐式信号只能逐步累积。
- 低强度隐式信号不创建强兴趣。
- 前端不发送 quick bounce；后端根据时长和进度派生。

## MVP 前端实现注意事项

- 使用现有 ArticleItem、Reader Action Bar、Popover / Bottom Sheet 模式，不新增复杂页面。
- 样式使用 CSS Modules + CSS Variables。
- 交互语义使用 React Aria Components。
- 图标按钮必须有 `aria-label` 和 tooltip。
- 负向动作在视觉上克制，不做大面积红色。
- 移动端核心动作触摸目标不小于 44px。
- 所有动作支持键盘触发。
- 操作按钮请求中必须防重复点击。
- 文章列表 mutation 采用乐观更新，但以 API 响应 state 为最终状态。
- `contentHtml` 阅读安全策略不在本文展开，继续遵守视觉规格文档中的正文渲染要求。

## 验收清单

- 每个指定动作都有触发位置、API type/value、视觉反馈、列表影响和推荐影响。
- `hide` 与 `not_interested` 成功后从当前普通列表移除。
- 收藏和稍后读在普通列表中不移除，在对应集合列表中取消后移除。
- 标记已读 / 未读在筛选列表中按筛选条件移除或保留。
- `open` 和 `read_progress` 不显示成功提示。
- 阅读进度只记录最高值，并按阈值或节流发送。
- 所有显式动作失败时回滚乐观更新。
- 文档未引入多用户、社交、标签、复杂设置页或恢复管理页。
