# 邸报 Dibao 低保真页面线框

## 文档目的

本文定义邸报 MVP 的低保真页面结构和核心交互。它不是视觉稿，不规定最终颜色、字体和细节样式。

线框目标：

- 确认页面信息架构。
- 确认桌面端和移动端的布局方向。
- 让后续设计系统和前端实现有共同骨架。

## 设计基调

邸报的界面应像一个安静、耐读、高信息密度的个人阅读器。

关键词：

```text
东亚阅读器
报纸感
低饱和
细边线
高文字密度
温润纸白
墨色层级
少卡片
少渐变
少装饰
```

避免：

```text
美式 SaaS dashboard
大圆角卡片堆叠
营销式 hero
紫蓝渐变
Bento Grid
```

## 交互规则

### 文章列表

- 点击标题或列表主体进入阅读页。
- 收藏、稍后读、不感兴趣必须可直接操作。
- 已读文章视觉弱化，但仍可访问。
- 推荐解释入口不应抢视觉焦点。

### 阅读页

- 打开文章记录 `open`。
- 滚动进度节流记录 `read_progress`。
- 达到阈值记录 `read_complete`。
- 用户显式操作优先于自动推断。

### 推荐解释

- 桌面端使用 Popover。
- 移动端使用 Bottom Sheet。
- 每篇文章最多展示 3～5 条原因。
- 不显示完整原始分数。

### 移动端

- 底部导航固定。
- 主要操作按钮尺寸适合触摸。
- 不依赖 hover。
- 阅读页底部工具条不能遮挡正文结尾。

## 线框验收标准

进入视觉设计和实现前，线框应满足：

- 首次设置可完整走通。
- 推荐、最新、阅读、订阅源、设置都有明确入口。
- 桌面和移动端主导航都明确。
- 推荐解释有桌面和移动端形态。
- 阅读设置能映射到 Reader Tokens。
- 没有营销式首页或装饰性 hero。
- 页面结构支持高信息密度阅读。
