# Dibao License Parameters

Dibao uses the Business Source License 1.1 to implement a source-available, fair-code, delayed-open-source licensing model.

## Long-Term Parameters

- License: Business Source License 1.1
- SPDX identifier: BUSL-1.1
- Licensor: Pls / Dibao project maintainer
- Commercial licensing contact: https://dibao.app
- Change License: Apache License 2.0
- Change License SPDX identifier: Apache-2.0
- Default Change Date Policy: each released version changes on the fourth anniversary of its first public release date.

## Additional Use Grant Summary

Before the Change Date, Dibao may be used in production for personal, household, family, research, evaluation, learning, non-commercial, self-hosted, and internal company or organization use. Paid consulting, deployment, training, migration, configuration, and operational support are allowed when the customer receives a Dibao instance in the customer's own environment, account, or controlled infrastructure.

Commercial hosting, SaaS, Managed Service, Cloud Service, white-label distribution, resale, competing commercial services, and commercial products where Dibao or modified Dibao is a core capability require a separate commercial license.

## Release Snapshots

Each release tag must contain a frozen `LICENSE.md` with the exact `Licensed Work`, `Release Date`, `Change Date`, and `Change License` for that version. `Release Date` is the date that specific version is first publicly released. That file is the legal snapshot for the release.

The `main` branch is a development branch and may contain placeholder dates or pending release parameters. For legal certainty, use the `LICENSE.md` in the relevant release tag.
